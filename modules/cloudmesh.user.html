cm_user Module¶

cloudmesh.user.cm_user¶

cm_user provides user information from the ldap and the cloud like openstack through mongodb. fab mongo.cloud command initialize mongo database and pour the information into the mongodb. Once the mongodb for cloudmesh has the user information, cm_user retrieves the user data from the mongodb instead of directly accessing the ldap and OpenStack Keystone. cm_user_id is the unique identification in ldap and cloud.

class cloudmesh.user.cm_user.cm_user[source]¶

Bases: object

cm_user provides user information including the ldap’s and the clouds’. The ldap has a user profile such as a first name, last name and active project ids. In OpenStack Keystone, it has cloud-related information such as a tenant id, user id, cloud version, cloud type and location.
