5   Cloud Mesh¶

A project to interface easily with multiple clouds from the commandline and a command shell.

5.1   Defaults¶

env

default ...

5.2   Register Clouds¶

Before you can use a cloud you need to register it. Registration will allow you to log into the cloud and use its resources.

5.2.1   Register an AWS account¶

cm register aws ...

Example:

  • Amazon AWS account

5.2.2   Register an azure account¶

cm register azure ...

Example:

  • Microsoft azure Cloud

5.2.3   Register an Eucalyptus account¶

cm register eucalyptus ...
  • FutureGrid india (Eucalyptus)

5.2.4   Register an OpenStack account¶

cm register openstack ...

Examples:

  • FutureGrid sierra
  • FutureGrid india (Openstack)
  • HP Cloud

5.2.5   Register an EC2 account¶

cm register ec2 ...

Examples:

  • FutureGrid alamo
  • FutureGrid hotel

5.3   Activate Clouds¶

cm activate <label>
cm deactivate <label>
::
cm activate <label> yes cm activate <label> no
::
cm activate <label> select

Prints a little menu that allows you to select a number and activate the cloud from the menu

6.2   cm manage initialize –user<name> –password <password>¶

  • connects to FG and retrieves important information for you to use
  • uses an ssh connection to connect to a FG service to retrieve the information
  • if the connection is refused n=5 times within the hour the
  • account is disabled
  • if the command is called more than 10 times a day the account is disabled

6.3   cm project list¶

lists the projects a user is in

6.4   cm project –info <number>¶

  • retrieves information about the project
  • if the user is the manager or lead it returns member information

6.5   cm project –activate <number>¶

  • activates charging for this project
  • subsequent calls are charged against this project

6.6   cm project –deactivate¶

  • deactivates any charge of a project

6.7   cm env¶

  • prints the environment and its variables currently associated with cm
